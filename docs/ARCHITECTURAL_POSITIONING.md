# Architectural Positioning — Site-First Re-Anchor

**Date:** 2026-05-25  
**Type:** Documentation and terminology alignment only — **no runtime logic changes.**

---

## Positioning statement

**Vilo OS is a Site Execution Operating System and Coordinator Survival OS.**

The product exists so **coordinators, sites, and PI/Sub-I** can execute protocol, prevent findings and deviations, reduce overload, protect revenue, and maintain operational continuity **before** external escalation.

Runtime intelligence is **site self-defense first**: finding prevention, operational explainability, and coordinator operational survival prioritization.

External parties (sponsor, CRO, CRA/monitor) may receive **Controlled External Visibility** only when the site authorizes **derived, scoped, delayed, operationally justified** outputs — never default raw runtime surveillance.

Vilo OS is **not** sponsor surveillance software, CRA-first tooling, or an oversight-first monitoring platform.
