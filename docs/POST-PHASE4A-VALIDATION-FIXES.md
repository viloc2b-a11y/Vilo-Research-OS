# Post–Phase 4A validation fixes

**When:** After migrations **0019** applied; Phase **4A** catalog GREEN; Phase **3C** GREEN; Phase **2** / **3B** initially RED.

**Outcome:** Failures were **not** caused by Phase **4A** schema or RLS regressions. Both issues were **fixture / validator correctness**.

---

## A. Root causes

| Suite | Symptom | Cause |
|-------|---------|--------|
| **Phase 2** | `attachments_policies_reference_org_membership` FAIL | **Validator drift:** `scripts/validate-phase2.mjs` read only `policyname` and `cmd` from `pg_policies`, then JSON-encoded those rows and searched for `user_organization_ids`. Policy expressions live in **`qual`** / **`with_check`**, so the substring never appeared — **false negative**. Attachment RLS in **`0011_attachments.sql`** was already correct. |
| **Phase 3B** | `complete_procedure_execution` returned *procedure completion is not allowed while visit status is locked* | **Broken fixture assumption:** Phase **3C** `lock_visit` / validation leaves the synthetic Phase **2** visit in **`visit_status = locked`**. Phase **3B** reset cleared **`procedure_executions`** and **`PROCEDURE_COMPLETED`** events but **did not unlock** the visit. **`complete_procedure_execution`** (0013) intentionally blocks completion when the visit is **`locked`** — correct RPC behavior, wrong harness state. **Not** related to **`source_definition_version_id`** or Phase **4A**. |

---

## B. Files changed

| File | Change |
|------|--------|
| `scripts/validate-phase2.mjs` | For **`attachments`**, select **`qual`** + **`with_check`** from **`pg_policies`**, concatenate, then assert **`user_organization_ids`** appears (same intent as before, correct signal). |
| `scripts/validate-phase3b.mjs` | After resetting procedure execution + events, **`UPDATE visits`** → **`visit_status = 'in_progress'`**, **`locked_at = null`**, **`locked_by_user_id = null`** for the fixture visit. Doc comment: prerequisites **0013** (not **0012**). |
| `docs/PHASE2-VALIDATION-RESULTS.md` | Regenerated by **`npm run db:validate-phase2`** (GREEN). |
| `docs/PHASE3B-VALIDATION-RESULTS.md` | Regenerated by **`npm run db:validate-phase3b`** (GREEN). |
| `docs/POST-PHASE4A-VALIDATION-FIXES.md` | This note. |

**No migration DDL changes.** No RPC / RLS weakening.
